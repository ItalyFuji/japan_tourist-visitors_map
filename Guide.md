## HTML（HyperText Markup Language）

**何者？**
ウェブページの「骨格」を作る言語。「ここにタイトルを置く」「ここに地図を置く」という構造を定義する。

**このアプリでの使い方**
`index.html` がアプリ本体。ヘッダー・地図エリア・フッターの配置をHTMLで組んでいる。

## CSS（Cascading Style Sheets）

**何者？**
ウェブページの「見た目」を決める言語。色・フォント・レイアウトを担当する。

**このアプリでの使い方**
`index.html` の `<style>` タグの中に書いている。ヘッダーの紺色、フッターのグレーなどを設定している。

## JavaScript

**何者？**
ウェブページに「動き」を付ける言語。ボタンを押したら何かが起きる、データを読んで表示する、といった処理を書く。

**このアプリでの使い方**
`index.html` の `<script>` タグの中に書いている。地図の描画やデータの読み込みはすべてJavaScriptで動いている。

## D3.js（Data-Driven Documents）

**何者？**
JavaScriptのライブラリ（便利な道具箱）。データをもとにグラフや地図などの視覚表現を作るのが得意。

**CDNとは？**
ライブラリをダウンロードしてフォルダに入れる代わりに、インターネット上から直接読み込む方法。
このアプリでは `https://cdn.jsdelivr.net/npm/d3@7` から読み込んでいる。

**このアプリでの使い方**
- 地図の描画（SVGのパスを生成）
- CSVファイルの読み込み（後で追加予定）
- 色のスケール計算（後で追加予定）

---

## SVG（Scalable Vector Graphics）

**何者？**
「数式で描く絵」の形式。拡大しても画質が劣化しない。地図・グラフの描画に使う。

**ピクセル画像との違い**
JPGやPNGは「点の集まり」で絵を表現するが、SVGは「この座標からこの座標まで線を引く」という命令で表現する。そのためどんな画面サイズでも綺麗に表示できる。

**このアプリでの使い方**
日本地図全体を1つのSVGとして描いている。都道府県の境界線は `<path>` タグ（複雑な形の線）として表現される。

---

## viewBox（ビューボックス）

**何者？**
SVGの「仮想キャンバス」のサイズを指定する仕組み。

**なぜ必要？**
SVGに `viewBox="0 0 800 700"` と書いておくと、「このSVGは800×700の仮想空間で描かれている」と宣言できる。
画面が小さくなっても、比率を保ったまま自動縮小してくれるので、スマホでも綺麗に表示できる。

---

## TopoJSON（トポJSON）

**何者？**
地図データのファイル形式。都道府県や国の「境界線の座標」が入っている。
GeoJSONという形式をさらに圧縮して軽量化したもの。

**このアプリでの使い方**
`https://raw.githubusercontent.com/dataofjapan/land/master/japan.topojson` から日本の都道府県境界データを読み込んでいる。

**topojson-client とは？**
TopoJSON形式のデータをD3.jsが扱える形式（GeoJSON）に変換するための道具。
`<script src="https://cdn.jsdelivr.net/npm/topojson-client@3">` で読み込んでいる。

## メルカトル図法（Mercator Projection）

**何者？**
地球（球体）を平面の地図に変換する方法のひとつ。Googleマップと同じ方式。

**なぜ変換が必要？**
地図データは「北緯35度・東経139度」のような緯度経度で表現されているが、画面は「右から何ピクセル・上から何ピクセル」というXY座標で表現されている。この変換を「投影」と呼ぶ。

**このアプリでの使い方**
`d3.geoMercator()` で日本を中心とした投影を作り、地図データをSVGの座標に変換している。

## インセット（Inset Map）

**何者？**
メインの地図の中に、別の小さな地図を埋め込む手法。天気予報で沖縄が左上に表示されるのと同じ。

**このアプリでの使い方**
沖縄県は本土から離れているため、本土地図を大きく表示しつつ、左上の枠内に沖縄を独立して表示している。
本土と沖縄で **別々の投影（座標変換）** を使っているのがポイント。

## 文字コード（エンコーディング）

**何者？**
文字をコンピュータ上で数値として表現する方式。日本語ファイルでよく問題になる。

**Shift-JIS と UTF-8**
- **Shift-JIS**：Windowsの日本語環境で昔から使われてきた方式
- **UTF-8**：現在の標準。ウェブやD3.jsはUTF-8が前提

**このアプリでの対応**
元の `city202603.csv` が Shift-JIS だったため、PowerShell で `city202603_utf8.csv` に変換した。
変換コマンド：
```powershell
$content = Get-Content 'city202603.csv' -Encoding Default
$content | Set-Content 'city202603_utf8.csv' -Encoding UTF8
```

## d3.rollup（データの集計）

**何者？**
D3の集計関数。データを「グループ化」して「まとめる」ことができる。

**このアプリでの使い方**
市区町村単位のCSVデータを都道府県ごとに合計している。
```
北海道の全市区町村の人数 → 合計 → 1つの数値
東京都の全市区町村の人数 → 合計 → 1つの数値
...
```
結果は `Map`（辞書）形式：`"北海道" → 1234567` のように都道府県名から人数を引ける。

## d3.scaleSequential（連続スケール）と interpolateReds

**何者？**
数値を色に変換する関数。

**仕組み**
`domain([最小値, 最大値])` で「最小値が一番薄い色、最大値が一番濃い色」と設定する。
`interpolator(d3.interpolateReds)` で「赤系グラデーション」を指定する。
→ 来訪者が多い都道府県ほど濃い赤、少ないほど薄い赤になる。

**domain を [0, 最大値] でなく [最小値, 最大値] にする理由**
全都道府県に必ず来訪者がいるため 0 から始めると色の差が潰れる。
実際のデータ範囲だけを使うことでグラデーションをフルに活用できる。

**interpolateReds(0.2 + 0.8 * t) にする理由**
素の `interpolateReds` は t=0 でほぼ白になる。
下限を 0.2 にずらすことで最小値でも「薄い赤」として視認できるようにしている。

## Promise.all（複数の非同期処理）

**何者？**
複数のデータ取得を「同時並行」で実行して、全部終わったら次の処理に進む仕組み。

**非同期とは？**
ウェブからデータを取ってくるときは時間がかかる。その間、ブラウザが止まらないよう「裏で取得しておいて、終わったら知らせてね」という処理方式を「非同期」と呼ぶ。

**このアプリでの使い方**
地図データ（TopoJSON）とCSVを同時に取得し、両方揃ったら描画を開始している。
```
Promise.all([地図データ取得, CSV取得]) → 両方完了 → 描画開始
```

## ツールチップ（Tooltip）

**何者？**
カーソルを要素に乗せたときに出てくる小さな情報ポップアップ。

**このアプリでの仕組み**
1. `<div id="tooltip">` という透明な箱をHTMLに用意しておく（初期状態は `display: none` で非表示）
2. D3の `.on("mouseover", ...)` でカーソルが乗ったときの処理を登録
3. カーソルが乗ったら `display: block` にして都道府県名を表示
4. `.on("mousemove", ...)` でカーソルの座標（`event.clientX/clientY`）を使い、ツールチップの位置をリアルタイムで更新
5. `.on("mouseout", ...)` でカーソルが離れたら再び非表示にする

**position: fixed とは？**
`position: fixed` を使うと、要素がスクロール位置に関係なく「画面上の絶対位置」に配置される。
`clientX/clientY`（画面上のカーソル座標）と組み合わせることで、ツールチップが常にカーソルのすぐ近くに表示できる。

**pointer-events: none とは？**
ツールチップ自身がマウスイベントを受け取らないようにする設定。
これがないと、カーソルがツールチップの上に来た瞬間に `mouseout` が発火してチップが消えてしまう。

## 人口データ（総務省 住民基本台帳）

**出典**
総務省「住民基本台帳に基づく人口、人口動態及び世帯数」（令和7年1月1日現在）
https://www.soumu.go.jp/main_sosiki/jichi_gyousei/daityo/jinkou_jinkoudoutai-setaisuu.html

**ファイル**
- `data/R6_population_prefecture.xlsx`：元データ（Excel形式）
- `data/population_pref.csv`：PowerShellで変換した加工データ（UTF-8 CSV）

**なぜ人口比率で色分けするのか**
来訪者数の絶対数で色分けすると東京など人口密集地が突出して、他の都道府県の差が見えなくなる。
人口比率（来訪者数 ÷ 人口 × 100）にすることで「その地域にどれだけ人が訪れたか」がより公平に比較できる。

## ローカルサーバー（開発中の動作確認）

**なぜ必要？**
`index.html` をダブルクリックして直接ブラウザで開くと `file://` プロトコルで動く。
このとき、ブラウザのセキュリティ機能がJavaScriptによるローカルファイル（CSVなど）の読み込みをブロックする。
ローカルサーバーを立てると `http://localhost` 経由で動くため、この制限を回避できる。

**起動方法（Python）**
プロジェクトフォルダで以下を実行：
```
python -m http.server 8080
```
その後ブラウザで `http://localhost:8080` を開く。
確認が終わったら `Ctrl+C` でサーバーを停止する。

**GitHub Pages では不要**
本番公開時は GitHub Pages が `https://` で配信するため、この問題は発生しない。あくまで開発中だけの手順。
