# 仕様書：日本 観光来訪者数マップ

最終更新：2026年5月7日

---

## 1. アプリ概要

日本の観光来訪者数をインタラクティブに可視化するウェブアプリ。
都道府県レベルのコロプレスマップに加え、年月選択・前年比/前月比比較・都道府県別の時系列グラフを備える。

- **対応データ期間**：2021年1月〜2026年3月
- **データ単位**：市区町村（可視化は都道府県単位で集計）
- **公開形式**：GitHub Pages（静的サイト。サーバー不要）

---

## 3. データソース

### 3-1. 観光来訪者数（メインデータ）

- **出典**：デジタル観光統計オープンデータ（一般社団法人日本観光振興協会）
- **URL**：https://www.nihon-kankou.or.jp/home/jigyou/research/d-toukei/
- **対象期間**：2021年〜2026年3月
- **単位**：市区町村

| 列名 | 内容 |
|---|---|
| 年 | 西暦年 |
| 月 | 月（1〜12） |
| 地域区分 | 市区町村 |
| データ区分 | 観光来訪者数 |
| 都道府県コード | 1〜47（整数） |
| 都道府県名 | 都道府県名称 |
| 地域コード | 市区町村コード（4〜5桁） |
| 地域名称 | 市区町村名 |
| 人数 | 来訪者数（人） |

**ファイル形式の違い：**
- 2021〜2025年：1ファイルに12ヶ月分すべて収録（年ファイル形式）
- 2026年：1ファイル1ヶ月（月ファイル形式）

### 3-2. 都道府県別人口

- **出典**：総務省「住民基本台帳に基づく人口、人口動態及び世帯数」
- **URL**：https://www.soumu.go.jp/main_sosiki/jichi_gyousei/daityo/jinkou_jinkoudoutai-setaisuu.html
- **対象年**：2021〜2026年（各年1月1日現在）
- **ファイル**：`data/population_pref_{year}.csv`（各年Excelから変換済み）

### 3-3. 都道府県境界（地図）

- **出典**：dataofjapan/land（GitHub）
- **URL**：https://raw.githubusercontent.com/dataofjapan/land/master/japan.topojson
- **形式**：TopoJSON（都道府県単位）

## 4. 実装済み機能

### 4-1. ① 都道府県コロプレスマップ

- 日本地図をSVGで表示
- 沖縄インセット：左上の枠内に独立した投影（126°E中心）で表示
- **右上コントロールパネル（画面固定）：**
  - **来訪者数 / 人口比トグル**：絶対数か人口100人あたりの比率かを切り替え
  - **年月セレクター**：2021年1月〜2026年3月を任意に選択。年変更時は最新月を自動選択
  - **前年比ボタン**：同月・前年との差を青（減少）〜グレー〜赤（増加）で表示
  - **前月比ボタン**：前月との差を同様の発散型色スケールで表示
- ホバーツールチップ：都道府県名・来訪者数・人口比率・比較中は変化率も表示
- 都道府県クリックで② に遷移

### 4-2. ② 都道府県別 時系列グラフ

- 都道府県をクリックすると表示（全期間データを初回のみロードしてキャッシュ）
- **縦棒グラフ：2021年1月〜2026年3月の推移**
  - デフォルト：全63ヶ月を月単位で表示（横スクロール対応）
  - 年単位ボタン：2021〜2025は年間合計に切り替え（計8本）
  - 来訪者数 / 人口比ボタン：Y軸の指標を切り替え
  - 年ごとに色分け（古いほど薄い赤 → 新しいほど濃い赤）
  - 年境界に区切り線とラベル
- ホバーで年月・来訪者数・人口比率を表示
- 「← 都道府県に戻る」で①に戻る

## 5. 色スケール仕様

### 通常モード（来訪者数 / 人口比）

| モード | スケール | ドメイン | 色 |
|---|---|---|---|
| 来訪者数 | `d3.scaleSequential` | 最小値〜最大値 | 薄い赤〜濃い赤 |
| 人口比 | `d3.scaleSequential` | 最小値〜最大値 | 薄い赤〜濃い赤 |

- `interpolateReds(0.2 + 0.8 * t)` を使用。下限を0.2にずらして最小値も視認できるようにする
- ドメインを `[0, 最大値]` でなく `[最小値, 最大値]` にすることで差を最大化

### 比較モード（前年比 / 前月比）

| 値 | 色 |
|---|---|
| 最大増加 | 濃い赤 `#ca0020` |
| 変化なし | グレー `#d0d0d0` |
| 最大減少 | 濃い青 `#2166ac` |

- `d3.scaleDiverging` with `d3.piecewise(d3.interpolateRgb, [青, グレー, 赤])`
- domain を `[Math.min(最小値, 0), 0, Math.max(最大値, 0)]` にすることで全増加・全減少でも崩れない

## 7. データ変換手順

原本CSVはShift-JIS、人口データはExcel形式のため、アプリが読める形に変換が必要。

```bash
cd japan_tourist-visitors_map
pip install openpyxl
python scripts/convert_data.py
```

変換内容：
- 観光CSV 8ファイル：Shift-JIS → UTF-8
- 人口Excel 6ファイル：都道府県合計行を抽出してCSV化

## 8. 開発環境セットアップ

```bash
git clone https://github.com/ItalyFuji/japan_tourist-visitors_map.git
cd japan_tourist-visitors_map
python -m http.server 8080
# ブラウザで http://localhost:8080 を開く
```

> `file://` プロトコルで直接開くとブラウザのセキュリティ機能がCSV読み込みをブロックする。

## 9. 来訪者数予測について

将来の来訪者数を予測する機能の実装可否についての見解。

### 実装面：技術的には可能

純粋なHTML+JSでも以下の手法は実装できる：

| 手法 | 難度 | 備考 |
|---|---|---|
| 線形回帰（トレンド直線） | 低 | D3のみで実装可 |
| 移動平均・指数平滑化 | 低〜中 | 同上 |
| 季節分解 | 中 | JSで実装可 |
| ARIMA | 高 | JSライブラリ要 |
| LSTM（深層学習） | 高 | TensorFlow.js要・重い |

### 予測精度面：現状のデータでは低〜中程度

以下の理由から、**参考値程度の精度にとどまる**と判断し、現バージョンでは実装しない：

1. **データ期間が短い**：2021〜2026年の約5年分。時系列予測には一般的に10年以上が望ましい
2. **COVID-19の歪み**：2021〜2022年は来訪者数が異常に少ない外れ値の年。そのままモデルに学習させるとトレンド推定が大きくズレる
3. **外部要因が強い**：自然災害・感染症・政府の観光施策・為替・大型イベント等、モデルが織り込めない要因が多い

### 実現可能な代替案

- **季節パターンの可視化**：月ごとの来訪傾向を視覚化（既に実装済みの月別グラフで確認可能）
- **回復トレンド表示**：2023〜2025年の直近トレンドを延長した参考線（実装は可能）
- **2019年以前データが入手できた場合**：COVID前後の比較ができ予測精度が向上する
