# 仕様書：日本 観光来訪者数マップ

## 1. アプリ概要

日本の観光来訪者数をインタラクティブに可視化するウェブアプリ。
デジタル観光統計オープンデータ（2026年3月・市区町村単位）を使用し、都道府県レベルのコロプレスマップで来訪者数を色分け表示する。

---

## 2. 技術スタック

| 項目 | 内容 |
|---|---|
| フレームワーク | なし（純粋な HTML + CSS + JavaScript） |
| 地図描画 | D3.js v7（CDN）|
| 地図データ形式 | TopoJSON（都道府県）/ GeoJSON（市区町村） |
| 公開方法 | GitHub Pages（静的サイト） |
| 開発環境 | Python `http.server`（ローカル確認用） |

---

## 3. データソース

### 3-1. 観光来訪者数
- **出典**：デジタル観光統計オープンデータ（一般社団法人日本観光振興協会）
- **URL**：https://www.nihon-kankou.or.jp/home/jigyou/research/d-toukei/
- **ファイル**：`data/city202603_utf8.csv`（元ファイルをUTF-8変換済み）
- **対象期間**：2026年3月
- **単位**：市区町村

| 列名 | 内容 |
|---|---|
| 年 | 2026 |
| 月 | 3 |
| 地域区分 | 市区町村 |
| データ区分 | 観光来訪者数 |
| 都道府県コード | 1〜47（整数） |
| 都道府県名 | 都道府県名称 |
| 地域コード | 市区町村コード（4〜5桁） |
| 地域名称 | 市区町村名 |
| 人数 | 来訪者数（人） |

### 3-2. 都道府県別人口
- **出典**：総務省「住民基本台帳に基づく人口、人口動態及び世帯数」（令和7年1月1日現在）
- **URL**：https://www.soumu.go.jp/main_sosiki/jichi_gyousei/daityo/jinkou_jinkoudoutai-setaisuu.html
- **ファイル**：`data/population_pref.csv`（Excelから加工・変換済み）

### 3-3. 都道府県境界（地図）
- **出典**：dataofjapan/land（GitHub）
- **URL**：https://raw.githubusercontent.com/dataofjapan/land/master/japan.topojson

### 3-4. 市区町村境界（地図）※実装検討中
- **出典**：smartnews-smri/japan-topography（GitHub）
- **URL**：https://raw.githubusercontent.com/smartnews-smri/japan-topography/main/data/municipality/geojson/s0010/N03-21_210101.json
- **備考**：全国一括ファイル（約5〜20MB）。都道府県別ファイルは不完全のため不使用。

## 4. 画面構成

### 4-1. ① 都道府県マップ画面（実装済み）

- 日本地図をSVGで表示（コロプレスマップ）
- 沖縄インセット：左上の枠内に独立した投影で表示
- 色分けモード（右上トグルスイッチで切り替え）：
  - **来訪者数モード**：来訪者数の多い都道府県ほど濃い赤
  - **人口比モード**：人口100人あたりの来訪者数で色分け
- ホバー時ツールチップ：都道府県名・来訪者数・人口比を表示
- 都道府県クリックで② に遷移

### 4-2. ② 市区町村画面（実装中・動作未確認）

**案A：市区町村コロプレスマップ**（現在実装中）
- クリックした都道府県の市区町村を地図で表示
- 来訪者数に応じた赤の濃淡で色分け
- ホバーで市区町村名・来訪者数を表示
- 「← 都道府県に戻る」で①に戻る

**案B：横棒グラフ**（実装済み・一時退避中）
- 市区町村を来訪者数順に横棒グラフで表示
- 来訪者数が多い順にソート
- ホバーで来訪者数を表示

## 6. 色スケール仕様

| モード | スケール種別 | ドメイン | 色 |
|---|---|---|---|
| 来訪者数 | `d3.scaleSequential` | 最小〜最大 | 薄い赤〜濃い赤 |
| 人口比 | `d3.scaleSequential` | 最小〜最大 | 薄い赤〜濃い赤 |

- `interpolateReds(0.2 + 0.8 * t)` を使用：下限を0.2にずらし、最小値でも薄い赤が視認できるようにする
- ドメインを `[0, 最大値]` でなく `[最小値, 最大値]` にすることでグラデーションをフルに活用

## 7. 地域コードの対応関係

CSVの `地域コード` とGeoJSONの `N03_007` の紐付けルール：

```javascript
// CSVの地域コード（4〜5桁）→ 5桁JISコードに変換
String(+csvRow["地域コード"]).padStart(5, "0")
// 例: 1101 → "01101"（札幌市中央区）
//    13101 → "13101"（東京都千代田区）
```

都道府県コードの2桁変換（市区町村GeoJSONフィルタ用）：
```javascript
String(+csvRow["都道府県コード"]).padStart(2, "0")
// 例: 1 → "01"（北海道）, 13 → "13"（東京都）
```

## 8. 開発環境セットアップ

```bash
# ローカルサーバーの起動（CSVをJavaScriptで読み込むために必要）
cd japan_tourist-visitors_map
python -m http.server 8080

# ブラウザで開く
http://localhost:8080          # 本番アプリ
http://localhost:8080/index_tokyo.html  # 東京都テスト
```

> **なぜローカルサーバーが必要か**
> HTMLファイルをダブルクリックで開くと `file://` プロトコルで動き、
> ブラウザのセキュリティ機能がローカルCSVファイルの読み込みをブロックする。
> `http://localhost` 経由なら制限なく動作する。
